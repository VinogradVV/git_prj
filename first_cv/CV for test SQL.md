
## 📊 Компоненты

### 1. Semantics (Семантический слой)
**Назначение**: Финальный слой, определяющий выходные поля и их свойства

**Поля семантического слоя**:

| Алиас | Наименование | Тип | Роль | Агрегация |
|-------|--------------|-----|------|-----------|
| `docdate` | Дата документа | `date` | Атрибут | - |
| `docnum` | Номер документа | `varchar` | Атрибут | - |
| `docyear` | Год документа | `integer` | Мера | Сумма |
| `id` | id строки | `test_tbl` | Атрибут | - |
| `shop` | Магазин | `varchar` | Атрибут | - |
| `shopname` | Название магазина | `varchar` | Атрибут | - |

### 2. Join_3 (Соединение)
**Тип**: Join (соединение таблиц)
**Назначение**: Объединение данных из двух проекций

**Входы**:
- Projection_1 (allinfo_tbl)
- Projection_2 (allinfo_tbl_sat1)

### 3. Projection_1 (allinfo_tbl)
**Назначение**: Основная проекция таблицы allinfo_tbl
**Источник**: `allinfo_tbl`

### 4. Projection_2 (allinfo_tbl_sat1)
**Назначение**: Дополнительная проекция таблицы allinfo_tbl_sat1 (SAT - таблица-сателлит)
**Источник**: `allinfo_tbl_sat1`

## 🔄 Поток данных

1. **Projection_1** и **Projection_2** извлекают данные из исходных таблиц
2. **Join_3** объединяет данные из обеих проекций
3. **Semantics** формирует финальный набор полей для потребителя

## 📝 Примечания

- Используется тестовая таблица `test_tbl`
- Поле `docyear` настроено как мера с агрегацией по сумме
- Остальные поля являются атрибутами без агрегации
